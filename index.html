About JSON Schema

JSON Schema specifies a JSON-based format to define the structure expected of JSON data for validation

JSON schema provides a contract for the JSON data required by a given application, and how that data can be modified. The schema is self-describing, as you may add a description field in each segment of your schema, as well as your schema as a whole. JSON Schema is currently in it's 4th version. You can find the full specifications and more examples at json-schema.org

Why ValidJSON?

One of the many uses of JSON Schema is validating REST service responses

In Front-End development, JSON responses are often expected of REST services provided by a Back-End. These responses must comform to a specific format. JSON Schema is a simple and effective way to ensure that these service responses are always formed as expected. Any deviation from what is specified in the schema will result in a detailed and controlled error.

ValidJSON.com is a quick and easy way to test JSON data (textarea on the right) against a Schema definition (textarea on the left). Prior to validation, your JSONs are checked, linted and prettified on focusout. You may choose from four pre-defined prettification styles in the menu.
